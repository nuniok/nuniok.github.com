服务端多版本支持

发表于 2010-01-01
  1. 服务端 API 区分版本号。
    1. /app/v2/xxx
    2. /app/v3/xxx
  2. API 与业务适当分离，通过扩展存在默认值的参数，升级接口。
  3. 开闭原则（扩展新接口，禁止修改原有接口）
  4. 客户端规范错误处理，怎加未来异常处理，引导用户升级。
  5. 监控 API 使用情况，废弃旧接口，引导用户升级。
